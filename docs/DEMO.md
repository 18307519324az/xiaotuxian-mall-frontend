# 前端演示路线

## 文档目的

本文用于说明前端公开演示时应该如何组织页面顺序、观察点和截图证据，让读者直接看到库存、权益和订单链路在页面层的完成度。

## 演示前准备

演示前建议确认：

- 前端开发服务器已启动
- 当前代理模式已经明确
- Mock Service 或真实服务链路已就绪
- 截图使用的页面状态没有残留旧订单或异常数据

推荐访问入口：

1. 首页：`http://localhost:8086/#/`
2. 分类页与库存排序：`http://localhost:8086/#/category/1005000`
3. 商品详情：`http://localhost:8086/#/goods/1369155859933827074`
4. 购物车：`http://localhost:8086/#/cart`
5. 结算页：`http://localhost:8086/#/member/checkout`

## 首页演示

首页演示重点在于说明项目不是零散页面集合，而是完整商城前台入口。公开展示时建议讲清：

- 顶部导航与分类入口
- 首页商品区与专题区
- 用户如何从首页进入分类页与详情页

对应截图：

- `docs/images/home.png`

## 分类页与库存排序

分类页是库存能力最直观的展示页面之一。演示时重点观察：

- 商品列表是否可以按库存排序
- “仅显示有货商品”是否可用
- 商品卡片上的库存信息是否清晰

对应截图：

- `docs/images/category.png`

## 商品详情与 SKU 库存

商品详情页需要说明 SKU 切换后的状态变化，而不是只停留在商品图片展示。推荐重点讲：

- SKU 选择
- 当前可售库存
- 低库存提示
- 立即购买与加入购物车动作是否受库存约束

对应截图：

- `docs/images/product-detail.png`

## 购物车库存校验

购物车页体现的是库存闭环的第二层防线。演示时可强调：

- 修改购买数量时会校验库存
- 批量勾选结算时会过滤异常状态
- 前端不会把明显超库存数量直接带到提交阶段

对应截图：

- `docs/images/cart.png`

## 结算页权益计算

结算页是权益联动最密集的页面。演示时重点观察：

- 优惠券选择后金额变化
- 礼品卡选择后金额变化
- 货到付款手续费联动
- 金额摘要区是否同步刷新

对应截图：

- `docs/images/checkout-benefit.png`

## 支付页权益摘要

支付页用于证明优惠券和礼品卡不是只在结算页临时显示。进入支付页后应继续确认：

- 应付金额
- 优惠券抵扣
- 礼品卡抵扣
- 订单编号与待支付状态

对应截图：

- `docs/images/pay-benefit.png`

## 订单详情页

订单详情页是验证权益快照是否真正落单的关键页面。演示时建议观察：

- 商品优惠字段
- 礼品卡抵扣字段
- 支付方式
- 收货信息与订单信息

如果使用当前仓库中的订单详情截图，可补充对比支付页与订单详情页之间的权益一致性。

## 异步下单状态轮询

前端仓不能只说“支持异步下单”，还要明确页面承担了状态轮询职责。建议说明：

- 异步下单发起后页面会先收到 `PROCESSING`
- 前端继续轮询订单处理状态接口
- 成功后进入支付页或订单详情页
- 失败时页面需要给出可理解的提示

这部分是前端仓与主仓异步链路文档形成闭环的关键说明。

## 常见问题

公开演示时最常见的问题包括：

- 代理仍指向错误服务
- 运行时订单数据残留导致页面状态异常
- 权益数据与当前演示账号不匹配
- 真实服务模式下依赖服务未全部启动

因此建议把页面截图制作和真实链路演示分开准备，避免相互干扰。
