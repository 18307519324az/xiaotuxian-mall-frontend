# 页面流程说明

## 首页到分类页

本文用于说明 PC 商城前端从首页到订单详情的主要页面流转，帮助读者快速理解页面之间的关系，以及库存、结算、支付和订单链路在前端是如何串起来的。

## 分类页到商品详情

首页承担平台的第一入口职责，主要展示：

- 顶部导航与搜索入口
- 主分类导航
- 轮播图和推荐区
- 品牌、专题和楼层商品

首页的价值不只是陈列内容，还承担把用户导向分类页、详情页和专题页的入口作用。

## 商品详情到购物车

分类页是库存展示和库存排序最直观的页面之一。当前页面支持：

- 分类筛选
- 商品卡片列表
- 按库存排序
- 仅看有货

这也是前端“库存闭环”最容易被读者看到的地方。

## 购物车到结算页

商品详情页负责展示：

- 商品图片与基础介绍
- SKU 选择
- 当前可售库存
- 低库存提示
- 加入购物车与立即购买

如果用户选中的 SKU 已无货，页面会限制购买动作，而不是等到下单时才暴露问题。

## 结算页到支付页

购物车页承担商品汇总与数量修改职责，当前流程包括：

- 勾选结算商品
- 修改购买数量
- 校验数量是否超过库存
- 进入结算页前再次确认商品状态

购物车页是库存校验从详情页延续到交易页的重要一环。

## 支付页到订单详情

结算页是前端交易逻辑最密集的页面之一，主要包含：

- 地址选择
- 配送时间选择
- 支付方式选择
- 商品清单与金额摘要
- 优惠券选择
- 礼品卡选择
- 货到付款手续费联动

这里同时也是同步下单和异步下单请求发起的入口。

## 会员中心订单流转

支付页主要承担订单提交后的信息确认与权益回显：

- 应付金额
- 优惠券抵扣
- 礼品卡抵扣
- 支付方式摘要
- 订单编号与支付状态入口

对于零元支付订单，页面也要能正确反映状态流转结果。

## 异步下单状态页

订单详情页负责展示：

- 商品列表
- 订单状态
- 收货信息
- 支付方式
- 优惠券和礼品卡快照
- 配送与订单扩展信息

它是验证“下单时的权益字段是否真的落到订单里”的关键页面。

## 异常状态与页面提示

会员中心是订单链路的延展区域，主要包含：

- 我的订单
- 收藏
- 浏览历史
- 地址管理
- 账户侧信息展示

对于公开展示来说，它说明前端并不只是单页下单，而是保留了完整用户侧流程。

## 页面之间的主链路补充

公开演示时常用的页面流转通常是：

```text
首页
  -> 分类页
  -> 商品详情页
  -> 购物车页
  -> 结算页
  -> 支付页
  -> 订单详情页
```

这条链路可以覆盖库存展示、库存校验、权益结算和订单结果回显。

## 为什么页面流程文档要单独保留

公开仓库读者通常不会先看源码路由，而是先看 README 和 docs。把页面流转单独写成文档，有两个直接价值：

1. 能快速说明前端覆盖的业务范围
2. 能帮助面试官把页面层能力与后端链路对应起来
